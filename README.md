# 乡村文化记忆库——基于AIGC的乡村数字记忆构建与传播平台

## 项目简介

本项目旨在利用 AIGC (生成式人工智能) 技术，通过数字化手段保存、构建和传播中国乡村的文化记忆。用户可以记录乡村故事、上传老照片，并利用 AI 技术进行修复和场景重构。

## 核心功能规划

### 1. 记忆采集与构建

- **多模态输入**：支持文字描述、图片上传、语音口述记录。
- **AIGC 故事增强**：通过 AI 将碎片化的口述记录润色为完整的文学叙事。
- **老照片 AI 修复/上色**：利用 AIGC 技术还原历史瞬间。
- **AI 场景生成**：根据文字描述生成乡村历史场景或非遗技艺展示图。

### 2. 数字记忆库

- **分类存储**：民俗、建筑、故事、手工艺、方言等。
- **时空地图**：结合地理位置和时间轴展示记忆点。

### 3. 多样化传播

- **虚拟展厅**：构建 3D/VR 乡村文化数字展馆。
- **社交分享**：支持生成精美海报、短视频片段进行社交媒体传播。

## 页面结构与各页详细设计方案

系统采用响应式设计，确保在 PC 端和移动端均有良好体验。

### 1. 首页 (Home - `/`)

* **Hero Section**：半透明毛玻璃导航栏；动态视频背景展示乡村节庆；大号书法标题；
* **"记忆图景"对比**：横向对比组件，用户手动拖动滑块查看老照片 AI 修复前后的惊艳对比。
* **核心功能入口**：通过大卡片引导至“数字长廊”、“创作中心”和“时空地图”。

### 2. 数字记忆长廊 (Gallery - `/memories`)

* **布局**：采用 **Masonry (瀑布流)** 布局，展示不同尺寸的 AI 修复图片与精美卡片。
* **分类筛选**：顶部包含“非遗、建筑、故事、民俗”标签，支持按地域搜索。
* **卡片设计**：悬浮加深阴影，显示“AI 增强”标识。

### 3. 记忆详情页 (Detail - `/memories/[id]`)

* **深度内容排版**：提供大字体、舒适行间距的阅读体验，配以传统宣纸质感纹理背景。
* **原图溯源**：在 AI 图片旁提供预览小窗，显示原始素材信息，体现 AI 的“重构”价值。
* **时空坐标**：底部显示该记忆发生的地理坐标和时间年代刻度。

### 4. AI 采集与创作中心 (Creative Studio - `/create`)

* **分步导航 (Stepper)**：
  * **Step 1: 素材提交**：上传图片、输入简单文字或录入方言语音（后端支持音频转文字）。
  * **Step 2: AI 工具库**：提供“文史专家（DeepSeek）”、“视觉画师（通义万相）”、“视觉修复（文心一格）”等工具卡片供用户选择。
  * **Step 3: 结果精修**：显示 AI 生成结果，允许用户微调文字或重新生成图片。
* **交互**：采用暗色调专业模式 UI，突出创作沉浸感。

## 交互体验优化 (UX Enhancement)

* **日间/夜间模式**：支持一键切换深色/浅色主题，适应不同光线下的阅读场景，深色模式采用“宣纸黑/水墨灰”配色。
* **骨架屏加载 (Skeleton Screen)**：在数据加载过程中展示优雅的骨架位位，消除白屏焦虑，提升感知速度。
* **微交互动画**：使用 Framer Motion 实现页面元素的淡入淡出、卡片悬浮缩放及平滑滚动效果，增加平台的高级感。

## 技术栈建议

- **前端**：Next.js / React (现代、SEO 友好)
- **后端 (核心枢纽)**：
  - **Go (Gin)**：作为高性能的主后端，采用业界主流的 Gin 框架，负责处理所有核心业务逻辑、高并发 AI API 调用、本地文件管理及 SQLite 数据库交互。
  - **Python (算法侧/可选)**：仅在需要调用特定 Python 图像处理库时作为辅助。
- **数据库**：Local SQLite (由 Go GORM 管理)
- **AI 接口**：**必须使用大赛指定的 15 款国产 AI 工具** (详见下方规范)
- **存储**：本地文件系统存储 (Local File System Storage)

## 后端与架构详细设计 (Go + Gin 驱动版)

采用 Go 语言结合 Gin 框架，不仅能提升系统的响应速度，还能体现出参赛者对工程质量和并发处理的卓越驾驭能力：

### 1. 架构方案

* **Gin 核心处理引擎**：
  * **模块化分层**：路由 (Router)、逻辑层 (Service)、数据访问层 (Repository)。
  * **高并发 AI 调度**：利用 Go 的 Goroutine 和 Channel，实现多款国产 AI 接口（如 DeepSeek 和通义万相）的异步并行调用。
* **高性能静态资源管理**：通过 Gin 实现对本地 `uploads/` 目录的高效访问。

### 2. 核心 API 路径预案

* `POST /api/v1/memories`：接收原始素材并触发 AI 工作流。
* `GET /api/v1/memories`：获取记忆库列表。
* `POST /api/v1/upload`：本地上传接口。

### 3. 数据持久化 (SQLite + GORM)

* **Memory (记忆条目)**：`id`, `title`, `category`, `rawContent`, `aiPolishedContent`, `location`, `timeScale`, `mediaPaths`.
* **Village (乡村实体)**：`name`, `history`, `tags`, `stats`.

## AI 工具合规规范

根据要求，非音乐类作品必须使用以下 10 款工具：

1. **阿里通义系列**：多模态生成、代码补全、可视化。
2. **百度文心系列**：多模态生成、深度学习（飞桨）。
3. **DeepSeek**：逻辑推理、复杂算法、长文本处理。
4. **稿定设计**：图文设计、海报/PPT 模板。
5. **和鲸 ModelWhale**：低代码 AI 开发、数据清洗。
6. **即梦、豆包 AI**：内容创作、视频编辑、多模态交互。
7. **科大讯飞星火**：语音处理、方言识别、智能配音。
8. **Kimi**：超长文本处理、文档分析。
9. **腾讯混元系列**：3D 资产生成、AI 编程、文本润色。
10. **智谱 AI**：技术文档、无代码 Bot 开发。

音乐类作品必须使用：ACE Studio, 海绵音乐, 腾讯音乐·启明星, 天谱, 网易云音乐·XStudio。

在 Windows 环境下，直接双击根目录下的 **`start.bat`** 即可同时启动前端 (Next.js) 和后端 (Go) 服务。

- 前端: `http://localhost:3000`
- 后端: `http://localhost:8080`
- 接口测试: `http://localhost:8080/ping`
